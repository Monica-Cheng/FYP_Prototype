# For Figma

This folder contains static HTML versions of interactive screens so they import
into Figma as full multi-screen layouts instead of only the first rendered
state.

For `S-01`, use:

- `S-01-Onboarding-Walkthrough-Figma.html`

That file renders all 3 onboarding cards side by side in one canvas with no
React and no button-driven state, which makes it suitable for HTML-to-Figma
import.

For `S-02`, use:

- `S-02-Login-Figma.html`

That file renders the main login screen plus its banner states as separate
static frames for import.

For `S-03`, use:

- `S-03-Register-Figma.html`

That file renders the default registration screen plus the validation-error
state as separate static frames for import.

For `S-04`, use:

- `S-04-Password-Reset-Figma.html`

That file renders the default password reset screen, invalid-email state, and
success state as separate static frames for import.

For `S-05`, use:

- `S-05-Onboarding-Body-Health-Figma.html`

That file renders the body profile onboarding screen in a blank/default state
and a connected/filled state as separate static frames for import.
